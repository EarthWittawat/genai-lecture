Foundations: Likelihood & the Universal Training Loop

A generative model learns the data distribution itself, then samples from it. The thread tying every later model together: the loss is a measure of "how wrong," and the same five-step loop minimizes it.

Generative goal
Model the data density
\[ p_\theta(x)\;\approx\;p_{\text{data}}(x) \]
  • Discriminative learns \(p(y\!\mid\!x)\) вҖ” predict a label.
  • Generative learns \(p(x)\) (or \(p(x,y)\)) вҖ” then sample new \(x\).
  • Goal: realistic and diverse novel samples.
Loss
Maximum likelihood = min cross-entropy
\[ \mathcal{L}(\theta)=-\mathbb{E}_{x\sim p_{\text{data}}}\big[\log p_\theta(x)\big] \]
  • Maximizing likelihood \(=\) minimizing the negative log-likelihood.
  • Classification вҶ’ cross-entropy; regression вҶ’ MSE.
  • A loss must be differentiable and scalar.
Training regime
The 5-step loop (PyTorch)
  1. Forward pass вҶ’ predictions
  2. Compute loss (the learning signal)
  3. Backward pass вҶ’ gradients via autograd
  4. Update parameters (optimizer step)
  5. Zero gradients, repeat
Step-by-step В· one optimization loop
CNN classifiers, GAN discriminators, diffusion score nets, LLM next-token вҖ” all run this loop. Only the architecture, loss definition, and data change.

Explain В· Q1

Likelihood & the learning signal

A generative model is said to "learn the data distribution."

  • (a) Contrast the generative objective \(p_\theta(x)\) with a discriminative objective \(p(y\!\mid\!x)\), and state what each lets you do.
  • (b) Show that minimizing cross-entropy is equivalent to maximum-likelihood estimation. Why must a loss be scalar and differentiable?
  • (c) Walk through the five-step training loop and say which step the loss equation actually feeds into.
(a) Generative vs discriminative

A discriminative model learns the conditional \(p(y\!\mid\!x)\): given an input, output a label вҖ” good for classifying/predicting but it cannot invent new \(x\). A generative model learns the full density \(p_\theta(x)\) (or joint \(p(x,y)\)); once you have \(p_\theta(x)\) you can sample brand-new, realistic, diverse examples on demand. Generative answers "what does the data look like?"; discriminative answers "given this data, what is it?"

(b) Cross-entropy = MLE

Maximum likelihood maximizes \(\sum_i \log p_\theta(x_i)\). Negating and averaging gives the cross-entropy between the data and the model, \(-\mathbb{E}[\log p_\theta(x)]\), so maximizing likelihood вүЎ minimizing cross-entropy. The loss must be a single scalar so we can rank "more wrong vs less wrong," and differentiable so autograd can compute \(\partial \mathcal{L}/\partial\theta\) for gradient descent.

(c) Where the loss enters the loop

Forward вҶ’ compute loss вҶ’ backward вҶ’ update вҶ’ zero. The loss equation is evaluated at step 2; its gradient (step 3) is what flows back to nudge \(\theta\) at step 4. The loss defines what "learning" even means вҖ” change the loss and you change the whole behavior of the same loop.

GANs: The Adversarial Minimax Game

Two networks play a zero-sum game: a generator turns noise into fakes, a discriminator judges real vs fake. No likelihood is computed вҖ” the discriminator is the learned loss function.

Generative
Noise вҶ’ data
\[ x = G(z;\theta_g),\quad z\sim p(z) \]
  • G maps a noise vector into data space.
  • D(x; Оё_d) outputs the probability that \(x\) is real.
  • Sampling is one forward pass of G вҖ” fast.
Loss
Minimax value function
\[ \min_{G}\max_{D}\; \mathbb{E}_{x\sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z\sim p(z)}[\log(1-D(G(z)))] \]
  • D maximizes: call real "real," fakes "fake."
  • G minimizes: make \(D(G(z))\to 1\) (fool D).
  • Implicitly minimizes JensenвҖ“Shannon divergence.
Training regime
Alternate two updates
  1. D-step: ascend on real + fake вҖ” sharpen the judge
  2. G-step: descend so fakes look real
  3. In practice G uses the non-saturating loss \(\max \log D(G(z))\)
  4. Repeat вҖ” a moving target for both
Step-by-step В· the adversarial loop
D and G are trained alternately. The same minimax loss is read with opposite signs: D climbs it, G descends it.
Math lenstwo gradients, one shared loss
Decompose the minimax

One equation, read twice

\[ V(D,G)=\underbrace{\mathbb{E}_{x}[\log D(x)]}_{\text{real term}} + \underbrace{\mathbb{E}_{z}[\log(1-D(G(z)))]}_{\text{fake term}} \]
  • D(x)probability D assigns to x being real, in (0,1) (sigmoid output).
  • G(z)fake sample from noise z вҲј p(z); a single forward pass through G.
  • log D(x)biggest when D(x)вҶ’1 вҶ’ D wants this on real data.
  • log(1вҲ’D(G(z)))biggest when D(G(z))вҶ’0 вҶ’ D wants this on fakes.

  • Non-saturating G loss: max log D(G(z)) вҶ’ strong gradient when fakes are still bad.
  • Optimal D recovers Jensen-Shannon divergence between p_data and p_G.

Explain В· Q2

The minimax objective & its training

Consider the vanilla GAN value function \(\min_G\max_D V(D,G)\).

  • (a) Write \(V(D,G)\) and explain each of its two terms from D's point of view and from G's point of view.
  • (b) Describe the alternating training regime. Why is G trained with the non-saturating loss \(\max_G\log D(G(z))\) instead of \(\min_G\log(1-D(G(z)))\)?
  • (c) Explain mode collapse by referring back to the loss вҖ” why does the equation permit it?
(a) The two terms
\[ V(D,G)=\mathbb{E}_{x\sim p_{\text{data}}}[\log D(x)]+\mathbb{E}_{z}[\log(1-D(G(z)))] \]

D wants to maximize: term 1 pushes \(D(x)\to 1\) on real data (\(\log D\) is largest near 1); term 2 pushes \(D(G(z))\to 0\) on fakes. G wants to minimize: G only touches term 2 and wants \(D(G(z))\to 1\) so \(\log(1-D(G(z)))\to-\infty\) вҖ” i.e. fool D. It's zero-sum: one's gain is the other's loss.

(b) Alternating + non-saturating G

Each iteration we take a gradient ascent step on D (sharpen the judge), then a gradient descent step on G (fool the sharpened judge). Early on, fakes are obviously fake, so \(D(G(z))\approx 0\); the term \(\log(1-D(G(z)))\) is then flat вҶ’ near-zero gradient for G (the saturation problem). Flipping to \(\max_G \log D(G(z))\) gives strong gradients exactly when G is losing, so G keeps learning.

(c) Why the loss allows mode collapse

The objective only asks G to make samples D cannot tell apart from real вҖ” it never rewards diversity. So G can find one "easy win" output that always fools the current D and produce only that, collapsing many modes into one. Because feedback comes solely from D (no likelihood term covering the whole distribution), nothing in the equation forces full coverage.

Advanced GANs: Conditioning & Cycle Consistency

Same adversarial backbone, two new ideas. Conditional GANs add a label \(y\) to both players. CycleGAN translates between unpaired domains using a cycle-consistency loss instead of paired data.

Generative
Condition & translate
\[ \text{cGAN: } x=G(z,y)\;\;\;\; \text{Cycle: } G\!:\!A\!\to\!B,\; F\!:\!B\!\to\!A \]

  • CycleGAN: two generators (\(G,F\)) + two discriminators (\(D_X,D_Y\)).

\[ \min_G\max_D \mathbb{E}_{x,y}[\log D(x,y)]+\mathbb{E}_{z,y}[\log(1-D(G(z,y),y))] \]

  • Cycle: \(F(G(a))\approx a\) and \(G(F(b))\approx b\).
  • О» trades style realism vs content preservation.

Adversarial + reconstruction

Explain В· Q3

From conditioning to cycle consistency

Both cGAN and CycleGAN extend the basic adversarial loss.

  • (a) Write the conditional minimax objective and explain the single change from vanilla GAN.
  • (b) Write CycleGAN's full objective and explain what the cycle-consistency term enforces and why it is needed for unpaired data.
  • (c) Explain the role of \(\lambda\): what happens at the two extremes?
(a) Conditional GAN
\[ \min_G\max_D \mathbb{E}_{x,y}[\log D(x,y)] + \mathbb{E}_{z,y}[\log(1-D(G(z,y),y))] \]

Structurally identical to vanilla GAN вҖ” the only change is that both terms are conditioned on \(y\). The label is fed to G (so it generates a specific class) and to D (so it judges "is this a real y?"). This makes generation controllable.

(b) CycleGAN objective
\[ \mathcal{L}=\mathcal{L}_{GAN}(G,D_Y)+\mathcal{L}_{GAN}(F,D_X)+\lambda\,\mathcal{L}_{cyc}(G,F) \]

Two GANs translate \(A\!\to\!B\) and \(B\!\to\!A\). The cycle term enforces \(F(G(a))\approx a\) and \(G(F(b))\approx b\) (an L1 reconstruction). With no paired data, the discriminators only guarantee the output looks like the target domain вҖ” they don't tie it to the specific input. The cycle constraint is what forces the translation to preserve content, ruling out arbitrary mappings.

(c) The weight О»

\(\lambda\) balances style realism vs content preservation. Too small вҶ’ the cycle barely matters, mappings become arbitrary (you lose the original content). Too large вҶ’ the network plays it safe and outputs something almost identical to the input (little translation happens).

Autoencoders & VAEs: The ELBO

A VAE makes the latent space probabilistic. Because \(\log p(x)\) is intractable, we maximize a lower bound (ELBO) = reconstruct well + keep the latent close to a prior. The reparameterization trick makes sampling differentiable.

Generative
Sample latent вҶ’ decode
\[ z\sim\mathcal{N}(0,I),\qquad x\sim p_\theta(x\mid z) \]
  • Encoder \(q_\phi(z\!\mid\!x)\) outputs \(\mu,\sigma\) вҖ” a distribution, not a point.
  • Reparameterize: \(z=\mu+\sigma\odot\epsilon,\;\epsilon\sim\mathcal{N}(0,I)\).
  • Decoder \(p_\theta(x\!\mid\!z)\) reconstructs \(x\).
Loss
Maximize ELBO (= min вҲ’ELBO)
\[ \mathcal{L}=\underbrace{\mathbb{E}_{q_\phi}[\log p_\theta(x\mid z)]}_{\text{reconstruction}}-\beta\,\underbrace{\mathrm{KL}\!\big(q_\phi(z\mid x)\,\|\,p(z)\big)}_{\text{regularize latent}} \]
  • Guarantee: \(\log p(x)\ge \text{ELBO}(x)\).
  • Recon вҶ’ fidelity; KL вҶ’ smooth, structured latent.
  • ОІ-VAE: higher ОІ вҶ’ more disentangled, blurrier.
Training regime
One joint optimization
  1. Encode \(x\) вҶ’ \((\mu,\sigma)\)
  2. Reparameterize to sample \(z\) (keeps gradients flowing)
  3. Decode вҶ’ \(\hat{x}\); compute recon + KL
  4. Backprop through both nets at once вҖ” stable, no minimax
Step-by-step В· the VAE pipeline
The trick: random \(\epsilon\) carries the noise, so the path \(x\!\to\!\mu,\sigma\!\to\!z\) stays differentiable and gradients reach the encoder.

\[ \mathrm{KL}\big(\mathcal{N}(\mu,\sigma^2)\,\|\,\mathcal{N}(0,1)\big)=\tfrac12\sum_i\big(\mu_i^2+\sigma_i^2-\log\sigma_i^2-1\big) \]

The ELBO & the reparameterization trick

A VAE is trained by maximizing the Evidence Lower Bound.

  • (a) Write the ELBO and name its two terms. What does each one push the model to do?
  • (b) Why do we optimize a lower bound at all instead of \(\log p(x)\) directly?
  • (c) State the reparameterization trick and explain precisely how it makes the training regime work. Why are VAE samples typically blurry?
(a) The ELBO terms
\[ \text{ELBO}= \mathbb{E}_{q_\phi(z\mid x)}[\log p_\theta(x\mid z)] - \mathrm{KL}\!\big(q_\phi(z\mid x)\,\|\,p(z)\big) \]

Reconstruction term rewards decoding \(z\) back into the original \(x\) (fidelity). KL term pulls the encoder's posterior toward the prior \(\mathcal{N}(0,I)\), giving a smooth, continuous, interpolatable latent space and preventing the encoder from cheating by spreading codes arbitrarily.

(b) Why a lower bound

\(\log p(x)=\log\int p_\theta(x\mid z)p(z)\,dz\) is intractable (integrating over all latents). The ELBO is a tractable quantity with the guarantee \(\log p(x)\ge\text{ELBO}\), so pushing the bound up pushes the true likelihood up too вҖ” and every term in the ELBO is computable and differentiable.

(c) Reparameterization & blurriness

Sampling \(z\sim\mathcal{N}(\mu,\sigma^2)\) is non-differentiable. We rewrite it as \(z=\mu+\sigma\odot\epsilon,\ \epsilon\sim\mathcal{N}(0,I)\): the randomness is moved into \(\epsilon\), so \(z\) is now a differentiable function of \(\mu,\sigma\) and gradients flow back through the encoder. This lets us train encoder + decoder jointly in one optimization (no minimax). Outputs are blurry because the Gaussian, pixel-wise-independent likelihood makes the decoder predict the mean under uncertainty, averaging plausible details together.

Recurrence, loss & BPTT

RNNs and LSTMs model sequences autoregressively.

  • (a) Write the recurrence \(h_t=f(h_{t-1},x_t;\theta)\) and explain how it lets the model factor \(p(x)\) over time.
  • (b) State the training loss and link it to the maximum-likelihood idea from Topic 1.
  • (c) Explain backpropagation through time and the sequential bottleneck. What problem does it create for long sequences, and how do LSTM gates address it?
(a) Recurrence вҶ’ factorized likelihood

Each step updates a hidden state \(h_t=f(h_{t-1},x_t;\theta)\), where \(h_t\) summarizes all inputs seen so far. Conditioning each prediction on \(h_{t-1}\) lets the model write the joint as a product of conditionals \(p(x)=\prod_t p(x_t\mid x_{autoregressive factorization.

(b) Loss
\[ \mathcal{L}=-\sum_t \log P(x_t\mid x_{

This is the summed next-token cross-entropy вҖ” exactly the negative log-likelihood of the sequence. Minimizing it = maximizing the probability the model assigns to the true sequence, the same MLE principle as Topic 1, applied step by step.

(c) BPTT & the bottleneck

We run the full sequence forward to accumulate the loss, then propagate gradients backward through every timestep (BPTT). Because \(h_t\) depends on \(h_{t-1}\), computation is strictly sequential вҖ” you can't skip or parallelize across time. Over long chains, repeated multiplication of Jacobians causes vanishing or exploding gradients, so early steps get little learning signal. LSTM gates (with an additive cell state) let information and gradients pass through largely unchanged, preserving long-term dependencies.

From the ELBO to the simple noise loss

DDPM defines a forward noising process and learns to reverse it.

  • (a) Write the closed-form forward equation for \(x_t\) and explain why "no learnable parameters" matters for training.
  • (b) Start from the variational lower bound and explain how it simplifies to \(\mathcal{L}_{\text{simple}}=\|\epsilon-\epsilon_\theta(x_t,t)\|^2\). What does the network actually learn to predict?

(a) Forward process
\[ x_t=\sqrt{\bar\alpha_t}\,x_0+\sqrt{1-\bar\alpha_t}\,\epsilon,\qquad \epsilon\sim\mathcal N(0,I) \]

The forward chain is fixed by the noise schedule \(\{\beta_t\}\) вҖ” it has no learnable parameters. This closed form lets us jump directly to any timestep \(t\) in one shot, so training never has to simulate the whole chain; we just sample a random \(t\) and corrupt \(x_0\) instantly.

(b) ELBO вҶ’ simple loss

The variational bound decomposes as \(L=L_T+\sum_{t>1}L_{t-1}+L_0\): \(L_T\) is parameter-free (\(x_T\) should be \(\mathcal N(0,I)\)); \(L_{t-1}\) is a KL between the learned reverse step and the true posterior \(q(x_{t-1}\mid x_t,x_0)\) вҖ” the main signal; \(L_0\) is a final reconstruction term. Ho et al. showed that re-parameterizing the reverse mean and dropping the weighting reduces \(L_{t-1}\) to a plain MSE between the true noise and predicted noise, \(\|\epsilon-\epsilon_\theta(x_t,t)\|^2\). So the network simply learns to predict the noise that was added.

(c) Training regime + DDIM/LDM

(1) Sample clean \(x_0\), timestep \(t\sim U\{1..T\}\), noise \(\epsilon\). (2) Build \(x_t\) via the closed form. (3) Feed \((x_t,t)\) to the U-Net to predict \(\epsilon\). (4) Take a gradient step on the MSE. It's stable вҖ” no adversary, no minimax. DDIM uses the insight that the training loss doesn't require a Markovian process, so sampling can skip timesteps for speed (same trained model). Latent Diffusion runs the identical loss and procedure, but on a frozen autoencoder's compact latent \(z\) instead of pixels вҖ” same loss, ~48Г— smaller tensors.

Explain В· Q8

Why LoRA trains so few parameters

LoRA adapts a large pretrained model without updating its weights.

  • (a) Write the LoRA reparameterization of a layer and explain the shapes of \(A\) and \(B\). Why is the parameter count \(r(d+k)\) instead of \(dk\)?
  • (b) The training loss is unchanged from full fine-tuning вҖ” so what exactly is different about the training regime? Trace what receives gradients.
  • (c) Explain QLoRA and why we can merge \(W+BA\) after training. Contrast with prompt engineering.
(a) The reparameterization
\[ W'=W+\Delta W=W+BA,\quad A\in\mathbb R^{r\times k},\ B\in\mathbb R^{d\times r},\ r\ll\min(d,k) \]

Instead of learning a full \(d\times k\) update (\(dk\) params), LoRA assumes the update is low-rank and factors it into two thin matrices. Trainable count is \(r\!\cdot\!k + d\!\cdot\!r = r(d+k)\), which is far smaller than \(dk\) when \(r\) is small (e.g. 8). The forward pass becomes \(h=Wx+BAx\).

(b) Same loss, different regime

The objective is still the task loss (e.g. language-model cross-entropy). What changes: the pretrained \(W\) is frozen (no gradients), and only \(A,B\) are trainable. Backprop computes \(\partial\mathcal L/\partial A\) and \(\partial\mathcal L/\partial B\) only; \(W\) stays static. So we get the full task gradient signal but update a tiny adapter вҖ” far less memory and compute.

(c) QLoRA, merging, vs prompting

QLoRA additionally quantizes the frozen backbone to ~4-bit (the trainable adapters stay in higher precision), letting much larger models fit on small hardware. Because \(W'=W+BA\) is just a sum, after training we can merge the adapter into the weights to get a single matrix with no extra inference latency. Versus prompt engineering вҖ” which is cheap, training-free, and good for formatting/instructions but doesn't change the model's parameters вҖ” LoRA changes behavior more deeply and is better for repeated, specialized tasks (at the cost of needing data + training).
